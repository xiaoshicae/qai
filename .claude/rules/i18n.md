# i18n 国际化规范

## 技术方案

- 使用 `react-i18next` + `i18next`
- 翻译文件：`src/locales/zh.json`（中文）、`src/locales/en.json`（英文）
- 配置文件：`src/i18n.ts`
- 默认语言：中文
- 持久化：localStorage (`qai-language`)

## 使用方式

### 组件内使用

```tsx
import { useTranslation } from 'react-i18next'

function Component() {
  const { t } = useTranslation()
  return <button>{t('dashboard.run_all')}</button>
}
```

### 带插值的翻译

```tsx
// zh.json: "confirm_delete": "确定删除「{{name}}」？"
t('common.confirm_delete', { name: '测试集A' })
```

### 切换语言

```tsx
import { changeLanguage } from '@/i18n'
changeLanguage('en') // 或 'zh'
```

## 翻译 key 命名规范

按功能模块分命名空间：

| 命名空间 | 说明 | 示例 |
|---------|------|------|
| `app.*` | 全局/应用级 | `app.name`, `app.select_collection` |
| `sidebar.*` | 侧边栏 | `sidebar.search`, `sidebar.history` |
| `dashboard.*` | Dashboard 页 | `dashboard.run_all`, `dashboard.add_case` |
| `scenario.*` | 场景表格 | `scenario.user_request`, `scenario.not_run` |
| `edit.*` | 编辑弹窗 | `edit.name`, `edit.save`, `edit.import_curl` |
| `chain.*` | 链式请求 | `chain.chain_name`, `chain.add_step` |
| `claude.*` | Claude Code 面板 | `claude.placeholder`, `claude.first_msg` |
| `settings.*` | 设置页 | `settings.language`, `settings.theme` |
| `tree.*` | 集合树 | `tree.add_request`, `tree.rename` |
| `runner.*` | 链运行等（保留命名空间） | `runner.run_chain` |
| `assertion.*` | 断言 | `assertion.status_code`, `assertion.eq` |
| `common.*` | 通用 | `common.delete`, `common.confirm_delete` |
| `group_menu.*` | 分组菜单 | `group_menu.new_suite` |
| `collection_menu.*` | 集合菜单 | `collection_menu.add_case` |

## 禁止事项

- **禁止在 JSX 中硬编码中文**（全部走 `t('key')`）
- **禁止在模块顶层常量中调用 `t()`**（`t()` 只能在 React 组件/hook 内调用）
  - 错误：`const TYPES = [{ label: t('xx') }]`
  - 正确：在组件内 `const TYPES = useMyOptions()`，自定义 hook 内调用 `t()`
- **禁止在非组件函数中使用 `t()`**，如果需要，改为参数传入：`function getItems(t: TFunction)`
- **禁止 `zh.json` 和 `en.json` 的 key 不一致**（必须同步添加）
