# 代码审查策略

## 核心方法（借鉴 Cursor Bugbot）

1. **边界情况优先** — 空值/零值/极大值、并发/竞态、网络失败/超时、非预期格式
2. **追踪副作用链** — 返回值消费方、错误是否静默吞掉、状态变更影响的依赖方
3. **动态上下文拉取** — 不只看 diff，拉取被调用函数签名、变量其他使用位置、类似模式
4. **攻击性探测** — 字符串比较→null混淆、条件分支→全路径覆盖、`unwrap()`→panic、SQL→参数化

## 必须报告 vs 忽略

**报告**：逻辑错误、边界 panic、安全漏洞、静默失败、数据丢失
**忽略**：纯风格问题、文档措辞、命名偏好（除非歧义）

## 逐变更检查

- [ ] 空/null 输入行为正确？
- [ ] 错误路径阻止后续执行？
- [ ] 返回值所有形态被处理？
- [ ] 外部调用失败正确传播？
- [ ] 并发/重复执行安全？
- [ ] 状态变更不影响依赖方？
