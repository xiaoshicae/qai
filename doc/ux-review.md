# UX 审查与优化记录 (2026-03-29)

## 审查方法

以专业 UED 视角，从以下维度全面审查：
- 信息层次与密度
- 交互流程与心智模型一致性
- 破坏性操作保护
- 空状态与反馈
- 组件一致性
- i18n 覆盖完整性
- 可访问性

---

## 已修复问题

### P0 — 影响功能正确性

#### 1. 删除确认弹窗模板语法错误

**问题**: `collection-overview.tsx` 中两处删除确认调用使用了错误的模板语法：
```tsx
// 错误：模板字符串嵌套 t() + ${name} 在对象中
confirm(`{t('common.confirm_delete', { name: ${name} })}`)

// 修复后
confirm(t('common.confirm_delete', { name }))
```

**根因**: 混淆了 JS 模板字符串和 i18n 模板参数。

**规范沉淀**: `ui-design.md` 新增 "i18n 规范"，禁止模板字符串嵌套 `t()` 调用。

---

#### 2. 编辑弹窗无「未保存提醒」

**问题**: 用户在编辑弹窗中输入内容后，点击遮罩层、Esc 或取消按钮，弹窗直接关闭，数据丢失。

**方案**:
- 打开弹窗时用 `useRef` 存储 JSON 快照
- 关闭前比较当前值与快照，有差异则弹确认框
- 新建模式下检测关键字段（name/url/body_content）是否已填写
- 提取 `closeEditDialog()` 统一处理三个关闭入口（遮罩/X/取消）

**规范沉淀**: `ui-design.md` 新增 "破坏性操作保护"，要求所有编辑弹窗实现快照比对。

---

### P1 — 影响核心体验

#### 3. 运行模式选择器不可见

**问题**: 并发/顺序模式、并发数、延迟配置全部藏在「运行全部」按钮的一个小三角后面。用户不知道能配置，改了也没反馈。

**方案**:
- 按钮旁显示当前模式简短标签（如 `⚡×5` 或 `≡ 3s`）
- Split Button 下拉区域作为按钮内部子元素（非独立按钮），避免视觉割裂
- 下拉面板使用 `glass-card` 统一风格

**规范沉淀**: `ui-design.md` 新增 "Split Button" 规范。

---

#### 4. 表格信息密度过高

**问题**: 7 列（SCENARIO / DESCRIPTION / EXPECT / STATUS / HTTP / TOTAL / 操作），DESCRIPTION 大多为空，EXPECT 和 HTTP 功能重叠。

**方案**: 精简为 5 列：
| 之前 | 之后 |
|------|------|
| SCENARIO | SCENARIO（不变） |
| DESCRIPTION | 移除（展开行中查看） |
| EXPECT | 移除（合并到 HTTP 列，无响应时显示期望值） |
| STATUS | STATUS（不变） |
| HTTP | HTTP（有响应显示实际值，无响应显示期望值灰色） |
| TOTAL | TIME（改名） |
| 操作 | 操作（不变） |

**规范沉淀**: `ui-design.md` 新增 "表格信息密度"，限制 ≤ 6 列。

---

### P2 — 影响一致性

#### 5. 空状态设计不统一

**问题**: 4 种不同的空状态风格（workbench 有图标+标题+副标题，history 只有一行文字，collection 又不同...）。

**方案**: 创建 `<EmptyState>` 组件：
```tsx
<EmptyState icon={Clock} title="暂无数据" description="补充说明" />
```
替换 history-view、runner-panel、chain-runner-panel、collection-overview 中的空状态。

**规范沉淀**: `ui-design.md` 新增空状态规范，禁止自行编写空状态 div。

---

#### 6. 按钮禁用态反馈不足

**问题**: `disabled:opacity-40` 太淡，无 `cursor-not-allowed`，用户点击无反应也不知道为什么。`sm` 尺寸 `h-7` 偏小。

**方案**:
- opacity 40 → 50
- 添加 `cursor-not-allowed`
- `sm` 尺寸从 `h-7` 统一到 `h-8`

**规范沉淀**: `ui-design.md` 新增按钮尺寸收敛规范（只允许 h-8 / h-9 两档）。

## 规范更新总结

本次审查向 `.claude/rules/ui-design.md` 新增了以下规范：

1. **破坏性操作保护** — 删除确认 + 编辑弹窗快照比对
2. **空状态** — 统一 `<EmptyState>` 组件
3. **按钮尺寸收敛** — h-8 / h-9 两档
4. **Split Button** — 下拉区域必须是按钮内部子元素
5. **表格信息密度** — ≤ 6 列，低价值列放展开行
6. **i18n 规范** — 禁止硬编码、禁止模板嵌套 t()
7. **组件检查清单** — 新增 EmptyState、ContextMenu、未保存提醒、禁用态等检查项
